## Initial Findings

- **Process Name:** `svohost.exe`
  
- **Process Path:** `C:\temp\service_installer\svohost.exe`

- **Parent Process:** `powershell.exe`

- **Command Line:** `\??\C:\Windows\system32\conhost.exe 0xffffffff -ForceV1`

- **File Hash:** `b432dcf4a0f0b601b1d79848467137a5e25cab5a0b7b1224be9d3b6540122db9`
  

  The first thing that caught my attention is wrong process name `svohost.exe`.It's suspicious because it closely called the legitimate `svchost.exe(Service Host)`.The second indicator is process path in `/temp` directory.Attacker often try to hide malicious file in this kind of accessible directory

## Conclusion

The evidence confirms that the Victor system was compromised via a PowerShell-based downloader, which executed a masqueraded payload (svohost.exe) and established a C2 connection. The malicious process remained active (PID 7640), indicating persistence and the potential for ongoing adversary activity. Immediate containment was performed, but full remediation (including system rebuild and credential rotation) is recommended.
